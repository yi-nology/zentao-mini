# Wails 接入项目 - 实现计划（分解和优先排序的任务列表）

## [x] 任务 1：安装 Wails 命令行工具
- **优先级**：P0
- **依赖**：None
- **描述**：
  - 安装 Wails v2 命令行工具
  - 验证安装是否成功
- **验收标准**：AC-1
- **测试要求**：
  - `programmatic` TR-1.1：执行 `wails version` 命令，确认版本信息正确显示
- **备注**：需要 Go 1.20+ 环境

## [x] 任务 2：初始化 Wails 项目
- **优先级**：P0
- **依赖**：任务 1
- **描述**：
  - 在项目根目录初始化 Wails 项目
  - 选择合适的项目模板（Vue.js）
  - 配置项目基本信息
- **验收标准**：AC-1
- **测试要求**：
  - `programmatic` TR-2.1：执行 `wails init` 命令后，检查生成的项目结构
  - `programmatic` TR-2.2：验证 wails.json 配置文件是否正确生成
- **备注**：初始化时选择 Vue 模板，以便与现有前端技术栈匹配

## [x] 任务 3：集成现有前端构建产物
- **优先级**：P0
- **依赖**：任务 2
- **描述**：
  - 将现有的前端构建产物（dist 目录）复制到 Wails 项目的前端目录
  - 调整前端资源路径配置
  - 确保前端能正确加载
- **验收标准**：AC-2
- **测试要求**：
  - `human-judgment` TR-3.1：启动 Wails 应用，验证前端页面是否正常显示
  - `human-judgment` TR-3.2：验证前端交互功能是否正常
- **备注**：可能需要修改前端的 API 请求基础 URL

## [x] 任务 4：集成现有 Go 后端代码
- **优先级**：P0
- **依赖**：任务 2
- **描述**：
  - 将现有 Go 后端代码（backend 目录）集成到 Wails 项目中
  - 调整后端服务的启动方式，使其在 Wails 应用中运行
  - 确保后端 API 接口可被前端访问
- **验收标准**：AC-3
- **测试要求**：
  - `programmatic` TR-4.1：验证后端服务是否成功启动
  - `programmatic` TR-4.2：测试 API 接口是否能正常响应请求
- **备注**：需要修改后端服务的监听地址和端口，使其在 Wails 应用内部运行

## [x] 任务 5：配置 Wails 应用
- **优先级**：P1
- **依赖**：任务 3, 任务 4
- **描述**：
  - 配置应用图标和名称
  - 设置应用窗口大小和位置
  - 配置应用的其他基本设置
- **验收标准**：AC-3
- **测试要求**：
  - `human-judgment` TR-5.1：验证应用图标是否正确显示
  - `human-judgment` TR-5.2：验证应用窗口设置是否合理
- **备注**：在 wails.json 中配置应用信息

## [x] 任务 6：测试 Wails 应用
- **优先级**：P0
- **依赖**：任务 3, 任务 4, 任务 5
- **描述**：
  - 运行 Wails 应用，测试所有功能
  - 验证前端和后端的集成是否正常
  - 检查应用性能和响应速度
- **验收标准**：AC-2, AC-3
- **测试要求**：
  - `human-judgment` TR-6.1：验证所有页面是否正常显示
  - `human-judgment` TR-6.2：验证所有功能是否正常工作
  - `programmatic` TR-6.3：测试 API 接口响应时间
- **备注**：重点测试前端与后端的交互功能

## [x] 任务 7：打包 Wails 应用
- **优先级**：P1
- **依赖**：任务 6
- **描述**：
  - 执行 Wails 打包命令，生成可执行文件
  - 测试打包后的应用是否能正常运行
  - 验证应用在目标平台上的兼容性
- **验收标准**：AC-4
- **测试要求**：
  - `programmatic` TR-7.1：执行 `wails build` 命令，确认打包成功
  - `human-judgment` TR-7.2：测试打包后的应用是否能正常启动和运行
- **备注**：根据目标平台选择合适的打包参数

## [/] 任务 8：优化和调整
- **优先级**：P2
- **依赖**：任务 7
- **描述**：
  - 优化应用启动时间和内存占用
  - 调整 UI 响应速度
  - 修复打包过程中遇到的问题
- **验收标准**：NFR-1, NFR-2, NFR-3
- **测试要求**：
  - `programmatic` TR-8.1：测量应用启动时间，确保不超过 3 秒
  - `programmatic` TR-8.2：测量应用内存占用，确保不超过 200MB
  - `human-judgment` TR-8.3：验证 UI 响应速度是否流畅
- **备注**：根据测试结果进行必要的优化
