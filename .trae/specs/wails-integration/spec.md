# Wails 接入项目 - 产品需求文档

## 概述
- **摘要**：将现有的前后端分离项目（Vue.js 前端 + Go 后端）接入 Wails 框架，构建为桌面应用程序。
- **目的**：通过 Wails 框架将 Web 应用转换为跨平台桌面应用，提供更好的用户体验和系统集成能力。
- **目标用户**：使用禅道系统的开发团队和项目管理人员。

## 目标
- 将现有的 Vue.js 前端和 Go 后端集成到 Wails 框架中
- 构建可在 Windows、macOS 和 Linux 上运行的桌面应用
- 保持现有功能完整性的同时，利用 Wails 提供的桌面特性
- 确保应用性能和用户体验达到桌面应用标准

## 非目标（范围外）
- 重写现有的前端或后端代码
- 添加新的业务功能
- 优化现有的业务逻辑
- 修改现有的 API 接口

## 背景与上下文
- 现有项目是一个前后端分离的禅道系统客户端，前端使用 Vue.js，后端使用 Go
- 前端已构建完成（dist 目录），后端提供完整的 API 服务
- Wails 是一个 Go 语言编写的桌面应用框架，允许开发者使用 Web 技术构建跨平台桌面应用

## 功能需求
- **FR-1**：将现有前端构建产物集成到 Wails 应用中
- **FR-2**：将现有 Go 后端服务集成到 Wails 应用中
- **FR-3**：实现 Wails 应用的基本配置和打包
- **FR-4**：确保应用能够在主流桌面平台上运行

## 非功能需求
- **NFR-1**：应用启动时间不超过 3 秒
- **NFR-2**：内存占用不超过 200MB
- **NFR-3**：UI 响应速度不低于现有 Web 版本
- **NFR-4**：支持 Windows 10+、macOS 11+ 和 Linux（Ubuntu 20.04+）

## 约束
- **技术**：使用 Wails v2 框架，Go 1.20+，Vue.js 3
- **业务**：保持现有业务逻辑不变
- **依赖**：现有项目的前端和后端代码

## 假设
- 现有前端代码已完全构建并可正常运行
- 现有后端代码可正常编译和运行
- 开发环境已配置好 Go 和 Node.js

## 验收标准

### AC-1：Wails 项目初始化
- **Given**：现有项目结构完整，前端已构建
- **When**：执行 Wails 初始化命令
- **Then**：成功创建 Wails 项目结构，包含必要的配置文件
- **Verification**：`programmatic`
- **Notes**：使用 wails init 命令初始化项目

### AC-2：前端集成
- **Given**：Wails 项目已初始化
- **When**：将现有前端构建产物复制到 Wails 项目的前端目录
- **Then**：前端资源正确集成，可在 Wails 应用中显示
- **Verification**：`human-judgment`
- **Notes**：验证前端页面能正常显示和交互

### AC-3：后端集成
- **Given**：Wails 项目已初始化，前端已集成
- **When**：将现有 Go 后端代码集成到 Wails 项目中
- **Then**：后端服务能在 Wails 应用中正常运行，API 接口可访问
- **Verification**：`programmatic`
- **Notes**：验证 API 调用能正常响应

### AC-4：应用打包
- **Given**：前后端已成功集成到 Wails 项目中
- **When**：执行 Wails 打包命令
- **Then**：成功生成可执行文件，可在目标平台上运行
- **Verification**：`programmatic`
- **Notes**：验证打包后的应用能正常启动和运行
